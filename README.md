LGS --- Digital Engineering Landing Page
======================================

A professional static landing page for LGS a software development company, built with Django and Tailwind CSS v4. Designed around the "Digital Architect" design system --- an editorial, asymmetric visual language that prioritizes tonal depth over decorative noise.

* * * * *

Getting Started
---------------

### Prerequisites

Ensure the following are installed on your machine:

-   [Python 3.12+](https://www.python.org/downloads/)
-   [UV](https://docs.astral.sh/uv/getting-started/installation/)
-   [Node.js LTS](https://nodejs.org/)

### Installation

**1\. Clone the repository:**

```
git clone https://github.com/yourusername/lgs-web.git
cd lgs-web

```

**2\. Install Python dependencies:**

```
uv sync

```

**3\. Install Node dependencies:**

```
npm install

```

### Running the Development Server

This project requires two processes running concurrently --- Django and the Tailwind CSS watcher.

**If you have `make` available (Git Bash / WSL / Linux / Mac):**

```
make dev

```

**Manually (two terminals):**

```
# Terminal 1 --- Tailwind watcher
npm run dev

# Terminal 2 --- Django development server
uv run python manage.py runserver

```

Visit `http://127.0.0.1:8000` in your browser.

### Build for Production

```
make static

```

This runs `npm run build` (minified Tailwind output) and `collectstatic` in a single command.

### Key Architectural Decisions

**`config/` instead of project name for settings** --- Unambiguous naming convention. The settings folder is always `config` regardless of the project name.

**Root-level `templates/` directory** --- All templates live at the project root, not inside the app. This makes template paths consistent and avoids app-namespacing issues for a single-app project.

**Partials pattern** --- Each page section is a self-contained partial prefixed with `_` (e.g. `_hero.html`). Global partials (navbar, footer) are included in `base.html`. Page-specific partials are included in the page template via `{% block content %}`.

**Context processors for section data** --- All dynamic content (nav links, methodology steps, testimonials, client names, footer links) is passed via context processors in `apps/core/context_processors.py`. Templates receive data --- they do not define it.

Design System Reference
-----------------------

This project follows the **"Digital Architect"** design system. The full specification is documented in `DESIGN.md`.

### Color Tokens

All colors are defined as CSS custom properties in `static/css/input.css` using Tailwind v4's `@theme` block and are available as utility classes throughout the project.

| Token | Value | Usage |
| --- | --- | --- |
| `primary` | `#E65100` | High-emphasis actions, deep branding |
| `primary-container` | `#F97316` | Hero CTAs, brand moments |
| `on-primary` | `#FFFFFF` | Text on orange backgrounds |
| `surface` | `#F7F9FB` | Base page background |
| `surface-container-low` | `#F2F4F6` | Alternating section background |
| `surface-container` | `#ECEEF0` | Nesting level 2 |
| `surface-container-lowest` | `#FFFFFF` | Elevated cards |
| `on-surface` | `#191C1E` | Primary text |
| `on-surface-variant` | `#44474E` | Secondary/muted text |

### Typography

**Font:** Manrope (Google Fonts) --- loaded via CDN in `base.html`.

| Role | Size | Usage |
| --- | --- | --- |
| Display Large | 3.5rem / 56px | Hero headlines |
| Headline Medium | 2.25rem / 36px | Section headings |
| Title Large | 1.375rem / 22px | Card titles |
| Body Large | 1rem / 16px | Body copy, line-height 1.6 |
| Label Medium | 0.75rem / 12px | Overlines, all-caps, +0.05em tracking |

### Core Rules

-   **No borders for sectioning** --- use surface color shifts between sections
-   **No rounded corners > 12px** --- architectural, not friendly
-   **No pure black** --- always use `on-surface` (`#191C1E`) for text
-   **Orange is a laser, not a floodlight** --- use `primary-container` only for CTAs and brand moments
-   **Left-heavy asymmetric grid** --- avoid center-aligning everything
